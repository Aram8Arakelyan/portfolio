I wanted a personal site that was fast, easy to maintain, and where publishing a
new post is as simple as adding a file. Here is how it is built.

## The stack

- **Next.js 16** with the App Router and React Server Components
- **Tailwind CSS v4** with **shadcn/ui** for a few polished primitives
- **MDX** files, committed straight to the repo — no database, no CMS

## Rendering posts

Each post is an `.mdx` file in `content/posts`. A small helper reads the folder,
parses the frontmatter, and returns the posts sorted by date:

```ts
import { getAllPosts } from '@/lib/posts'

const posts = await getAllPosts()
const latest = posts.slice(0, 3)
```

Individual posts are statically generated with `generateStaticParams`, so every
article is plain HTML served from the edge. Syntax highlighting happens at build
time, which means there is zero client-side JavaScript shipped just to colour code.

## Files vs. a CMS

| Approach          | Pros                              | Cons                    |
| ----------------- | --------------------------------- | ----------------------- |
| Files in the repo | Versioned, fast, offline-friendly | You edit in your editor |
| Headless CMS      | Nice editing UI                   | Extra service to run    |

For a personal blog, files win. The whole site is in one place and deploys as a
static bundle.
